# windowDrive_Core 🚀

Автоматизированный программно-аппаратный комплекс на базе **ESP32 (Wemos D1 Mini 32)** для интеллектуального координатного управления мансардными окнами по Wi-Fi.

---

## 🛠️ Аппаратная архитектура и GPIO

* **Микроконтроллер**: ESP32 (двухъядерный чип, работа в цикле Ядра 1).
* **Силовые ключи драйвера**: Двухмостовая схема управления реверсом.
  * `PIN_IN1_HIGH` (GPIO 16) — Верхнее левое плечо (ШИМ-канал LEDC 0).
  * `PIN_IN2_HIGH` (GPIO 17) — Верхнее правое плечо (ШИМ-канал LEDC 1).
  * `PIN_IN1_LOW`  (GPIO 18) — Нижнее левое плечо (Digital Output).
  * `PIN_IN2_LOW`  (GPIO 19) — Нижнее правое плечо (Digital Output).
* **Обратная связь**: Аппаратный квадратурный энкодер (модуль PCNT).
* **Экстренный сброс**: Физическая кнопка очистки памяти Wi-Fi на GPIO 22.

---

## 🧠 Ключевой функционал силового ядра

### 1. Интеллектуальная двухуровневая защита по току
* **Маршевый автостоп (50 мс)**: Непрерывное вычисление скользящего среднего тока шунта строго по исторической части (прошлые 2/3 накопительного массива). Динамическая отсечка аномальных выбросов при заклинивании створки с учетом коэффициента чувствительности.
* **Абсолютный потолок АЦП**: Жесткий мгновенный стоп с 1-й миллисекунды хода, защищающий редуктор при глухом ударе в раму на старте.
* **Тайм-аут редуктора**: Аварийная отсечка при непрерывном движении более 2 минут.

### 2. Адаптивный конечный автомат (motor_tick)
* **Координатное ведение**: Автоматическое позиционирование окна по слайдеру с коридором упреждения инерции в 10 шагов.
* **Силовой дожим**: Координатное ведение отключается в крайних точках (0% и 100%) — запуск силового наката до физического упора в уплотнитель для герметичности.
* **Защита от встречных токов**: Жесткая неблокирующая пауза реверса (1000 мс) для гашения выбега ротора при переключении направлений на лету.

### 3. Энергонезависимая память
* Автоматическое сохранение и чтение 6 ячеек быстрой памяти положений (пресеты M1-M6) в EEPROM.
* Хранение калиброванных границ и параметров железа во Flash-памяти через библиотеку `Preferences`.

## 🌐 Веб-интерфейс и сетевые службы

* **Главный пульт (`/`)**: Пастельно-темный адаптивный интерфейс с кастомной PNG-иконкой мансардного окна. Реализована плавная SVG-анимация хода створки. Интегрирован скрипт блокировки отскоков ползунка слайдера при перетаскивании пальцем на мобильных устройствах. Ссылка на настройки скрыта от гостей.
* **Инженерное меню (`/setting`)**: Доступно по прямому секретному адресу. Позволяет на лету регулировать мощность ШИМ (рабочий безопасный диапазон 50–230), чувствительность автостопа и порог АЦП удара.
* **Бесшовное обновление (`/update`)**: Встроенная служба асинхронного OTA-обновления бинарными файлами прошивки.
* **Сетевая служба**: Локальное сетевое имя платы через mDNS (`http://okno.local`).

### 📡 Кастомный Каптивный портал (Первичная настройка)
При отсутствии сохраненной сети контроллер поднимает точку доступа `windowDrive_Core` (192.168.4.1). Форма ввода параметров сети обрабатывается через фоновый AJAX-запрос:
1. Пользователь вводит SSID/Пароль и нажимает «Сохранить настройки».
2. Форма **тихо отправляет данные в фоне без перезагрузки страницы**.
3. На экране выводится финальная подсказка, а страница **замирает и остается полностью рабочей** перед глазами пользователя:
> *После сохранения настроек и подключения устройства к домашней сети перейдите по этой ссылке либо скопируйте её в адресную строку браузера: http://okno.local*
4. Плата удерживает соединение 1 секунду для гарантированной отправки пакетов в браузер и уходит в автоматический рестарт (`ESP.restart()`).
